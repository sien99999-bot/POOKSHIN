# 🥐 POOKSHIN 모바일 웹 구현 가이드 (Claude Code용)

이 문서는 **AI 코딩 어시스턴트(Claude Code 등)** 환경에서 POOKSHIN 앱을 **HTML/CSS 기반 모바일 웹**으로 구현하기 위한 가이드입니다.

---

## 0. 공통 규칙

아래 규칙은 모든 STEP에 공통 적용한다.

### 공통 지시문
매 단계 프롬프트 맨 위에 아래 문장을 반드시 포함해서 사용한다.

```text
반드시 HTML/CSS 기준으로 작성해줘.
모바일 웹 화면 기준으로 구현해줘.
React Native / Expo 문법은 사용하지 마.
Figma 디자인과 제공된 이미지를 함께 참고해서 최대한 비슷하게 구현해줘.
이미지는 로컬 images 경로 기준으로 연결해줘.
한 번에 너무 많이 만들지 말고, 이번 단계 범위만 작업해줘.
```

### 기술 기준
- **구조**: `index.html` (온보딩), `home.html` (홈), `detail.html` (상세), `map.html` (지도), `my.html` (마이페이지), `setting.html` (설정)
- **스타일**: 각 html 문서별 전용 CSS 파일 작성 (예: `home.css`, `my.css`, `detail.css` 등)
- **동작**: 필요한 경우 `script.js` 작성 (프레임워크 없이 순수 JS 사용)
- **이미지**: `public/images/` 폴더 내 로컬 파일만 사용 (`images/파일명`으로 참조)

### 이미지 규칙
- 모든 이미지는 `images/` 하위 로컬 파일만 사용 (외부 URL 금지)
- Figma asset URL 직접 사용 금지
- 이미지가 없는 경우 `placeholder` 박스로 처리

---

### **STEP 1: 초기 설정 및 온보딩 01~02**
> **(공통 지시문 필수 포함)**
> "Vite 기반 프로젝트 구조를 잡고, `index.html`과 `index.css`를 만들어줘. `화면이미지/스플래쉬.png`, `온보딩01.png`, `온보딩02.png`를 참고해 배경색(#F8F4EC)과 `images/` 폴더의 에셋을 배치해줘. 모바일 화면 중앙 정렬 레이아웃을 잡아주는 게 핵심이야."

### **STEP 2: 취향 선택 온보딩 03~05**
> **(공통 지시문 필수 포함)**
> "사용자의 빵 취향을 선택하는 `index.html`의 다음 섹션들을 구현해줘. `화면이미지/온보딩03, 04, 05.png`를 참고해 빵 종류 그리드, 해시태그 칩 선택 UI, 그리고 상단 점(Dots) 인디케이터를 CSS로 정교하게 만들어줘."

### **STEP 3: 소셜 로그인 화면**
> **(공통 지시문 필수 포함)**
> "`화면이미지/로그인.png` 디자인대로 카카오, 구글, 애플 로그인 버튼을 구현해줘. 버튼의 둥근 모서리, 배경색, 로고 아이콘 배치를 피그마 시안과 동일하게 맞춰줘."

### **STEP 4: 홈 추천 화면 (Home)**
> **(공통 지시문 필수 포함)**
> "`home.html`과 `home.css`를 생성해줘. `화면이미지/홈 - 추천화면.png`를 분석해서 상단 헤더, 와앙이 인사말, 중앙의 제품 카드(크루아상), 하단의 NEXT/POKI 버튼을 배치해줘. 탭바(Navigation)도 하단에 고정해줘."

### **STEP 5: 제품 상세 페이지 (Detail)**
> **(공통 지시문 필수 포함)**
> "`detail.html`과 `detail.css`를 생성해줘. `화면이미지/상세페이지.png`를 참고해 Baker's Note, 식감 데이터 바(5단계 세그먼트), 도넛 차트 시각화 요소를 구현해줘."

### **STEP 6: 포키 완료 및 길찾기 유도**
> **(공통 지시문 필수 포함)**
> "`화면이미지/포키완료페이지.png` 디자인을 구현해줘. 체크마크 아이콘과 '길 찾기' 버튼이 포함된 간결한 피드백 화면이야."

### **STEP 7: 지도 기반 베이커리 탐색 (Map)**
> **(공통 지시문 필수 포함)**
> "`map.html`과 `map.css`를 생성해줘. `화면이미지/지도.png`를 참고해 상단 지도 영역과 하단 베이커리 리스트 카드를 구현해줘. 리스트에는 가게명, 주소, 별점, 북마크 아이콘이 들어가야 해."

### **STEP 8: 리뷰 목록 및 작성 모달**
> **(공통 지시문 필수 포함)**
> "`review.html` 또는 상세페이지 내 리뷰 섹션을 구현해줘. `화면이미지/리뷰페이지.png`의 평점 요약과 리뷰 리스트, 그리고 `리뷰하기.png`의 별점/식감 선택 폼을 만들어줘."

### **STEP 9: 마이페이지 및 취향 DNA 모달**
> **(공통 지시문 필수 포함)**
> "`my.html`과 `my.css`를 생성해줘. 사용자의 '내 취향 데이터' 버튼 클릭 시 나타나는 모달 창을 만들고, 레이더 차트와 트렌드 그래프를 CSS/SVG로 구현해줘."

### **STEP 10: 설정 및 최종 폴리싱**
> **(공통 지시문 필수 포함)**
> "`setting.html`과 `setting.css`를 생성해줘. 메뉴 목록과 토글 스위치 인터랙션을 구현하고, 전체 화면 간의 링크 연결과 부드러운 전환 효과를 적용해줘."
